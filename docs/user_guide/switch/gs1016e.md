### II. Port Management
- [Port Settings](port_management.md#port-settings){ data-preview } configures the Ports' parameters, including the state, auto-negotiation, data transfer rate and mode, as well as the flow control. 
- [Port Trunk](port_management.md#port-trunk){ data-preview } allows multiple ports to be bundled together to increase bandwidth and provide redundancy data for transmission.
- [Port Mirror](port_management.md#port-mirror){ data-preview } enables the traffic of one port to be copied to another port for monitoring and analysis purposes.
- [Speed Limit](port_management.md#speed-limit){ data-preview } restricts the data transmission rate of a port to prevent network congestion and ensure fair bandwidth allocation. 
- [Jumbo Frame](port_management.md#jumbo-frame){ data-preview } supports the transmission of larger data frames (typically up to 9,000 bytes) to improve network efficiency and reduce overhead. 
- [EEE Config](port_management.md#eee-config){ data-preview }(Energy-Efficient Ethernet configuration) allows the switch to reduce power consumption during periods of low data activity while maintaining network connectivity. 
- [Storm Control](port_management.md#storm-control){ data-preview } limits the number of broadcast, multicast, or unknown unicast frames on a port to prevent network storms that can overwhelm the network. 
- [Traffic Monitor](port_management.md#traffic-monitor){ data-preview } monitors network traffic to identify patterns, detect anomalies, and provide insights into network performance and usage. 
- [MAC Manage](port_management.md#mac-manage){ data-preview } manages MAC addresses by learning, filtering, and controlling access based on MAC addresses to enhance network security and manageability. 

### III. Protocol Management
- [SNMP](protocol_management.md#snmp){ data-preview } is a widely used protocol for managing and monitoring network devices. It allows network administrators to collect information from and configure devices such as routers, switches, and servers. 
- [IGMP Snooping](protocol_management.md#igmp){ data-preview } is a technique used by switches to monitor IGMP messages between hosts and routers. It helps optimize multicast traffic by ensuring that multicast packets are only forwarded to ports where interested receivers are present. 
- [DHCP Snooping](protocol_management.md#dhcp-snooping){ data-preview } is a security feature used on switches to prevent unauthorized DHCP servers from assigning IP addresses to devices on the network. It helps protect against IP address spoofing and ensures that only legitimate DHCP servers can operate.
- [Loop Prevention](protocol_management.md#loop-prevention){ data-preview } is used in networks to avoid the creation of loops that can cause broadcast storms and network instability. Techniques such as Spanning Tree Protocol (STP) are commonly used to detect and prevent loops in Layer 2 networks.
- [Spanning Tree](protocol_management.md#spanning-tree){ data-preview } (STP) is a network protocol that prevents loops in a network by ensuring that there is only one active path between any two network devices. It achieves this by creating a loop-free topology using a tree structure.   
- [LLDP Config](protocol_management.md#lldp-config){ data-preview } is a protocol used to exchange information between adjacent network devices. It helps in discovering device information, such as device type, port details, and VLAN information, which can be useful for network management and troubleshooting. 
- [LLDP Neighbor](protocol_management.md#lldp-neighbor){ data-preview } displays information about adjacent devices discovered via LLDP.

### IV. [VLAN](vlan.md#vlan){ data-preview } 
- [MTU VLAN](vlan.md#mtu-vlan){ data-preview } defines an uplink port which will build up several VLANs with each of the other ports. Each VLAN contains two ports, the uplink port and one of the other ports in the switch, so the device connected to the uplink port can communicate with the device connected to any other port, but devices connected to other ports cannot communicate with each other.
- [Port-based VLAN](vlan.md#port-based-vlan){ data-preview }s are divided based on ports. In port based VLAN mode, each port can only be added to one VLAN.
- [802.1Q VLAN](vlan.md#8021q-vlan){ data-preview } protocol defines a new format of VLAN data frame (Tagged Frame). As the following figure shows, compared to the traditional Ethernet data frame (Untagged Frame), the VLAN data frame (Tagged Frame) adds a VLAN tag.
- [Voice VLAN](vlan.md#voice-vlan){ data-preview } is dedicated specifically for IP phone traffic, often prioritizing it (via QoS) and separating it from data traffic for better quality and security.
