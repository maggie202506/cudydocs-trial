### IV. WAN
- [Bridged Mode](pon_wan.md#bridged-mode){ data-preview }: Passes raw traffic transparently (no NAT/auth), pushing all routing tasks to downstream devices.
- [IPoE Mode](pon_wan.md#ipoe-mode){ data-preview }: Automatically assigns IP via DHCP without authentication, ideal for plug-and-play services like IPTV.
- [PPPoE Mode](pon_wan.md#pppoe-mode){ data-preview }:Requires username/password authentication to establish a connection, commonly used for home broadband.

### V. Services

[**Service**](pon_service.md){ data-preview }

- [DHCP](pon_service.md#dhcp){ data-preview } is to dynamically assigns IP addresses to devices on your local network for internet communication.
- [Dynamic DNS (DDNS)](pon_service.md#dynamic-dns){ data-preview } supports automatically updating a domain name with your router's changing public IP address, allowing stable remote access. 
- [IGMP Proxy](pon_service.md#igmp-proxy){ data-preview } relays multicast group membership information (like for IPTV streams) between your local network and the ISP network.
- [UPnP](pon_service.md#upnp){ data-preview } allows compatible devices on your network (like games or media servers) to automatically configure router port forwarding rules for easier connectivity.
- [RIP](pon_service.md#rip){ data-preview } is a basic routing protocol where routers periodically share their entire routing tables to learn network paths (less common on home routers).

[**Firewall**](pon_service.md#ipport-filtering){ data-preview }

- [IP/Port Filtering](pon_service.md#ipport-filtering){ data-preview } blocks or allows specific network traffic based on its source/destination IP address and/or port number. 
- [MAC Filtering](pon_service.md#mac-filtering){ data-preview } restricts network access to only devices with specific, pre-approved hardware (MAC) addresses.  
- [Port Forwarding](pon_service.md#port-forwarding){ data-preview } redirects incoming internet traffic on a specific port to a designated device/server within your private network. 
- [URL Blocking](pon_service.md#url-blocking){ data-preview } prevents access to specific websites based on their web address (URL) or keywords within the address. 
- [Domain Blocking](pon_service.md#domain-blocking){ data-preview } prevents access to all websites belonging to a specific internet domain name. 
- [DMZ](pon_service.md#dmz){ data-preview } exposes one specific device on your network directly to the internet (bypassing most firewall protections) for hosting services, increasing security risks. 

### VI. Advance

[**Advance**](pon_advance.md#advance-1){ data-preview }

- [ARP Table](pon_advance.md#arp-table){ data-preview } displays a list of learned MAC addresses using the IP addresses. 
- [Bridging](pon_advance.md#bridging){ data-preview } allows to configure the bridge parameters as well as its attached ports. 
- [Routing](pon_advance.md#routing){ data-preview } is to configure the IP routes. 
- [Link Mode](pon_advance.md#link-mode){ data-preview } is to set the Ethernet link speed and duplex mode. 
- [Print Server](pon_advance.md#print-server){ data-preview } enables network printing by sharing a locally connected USB printer.

[**IP QoS**](pon_advance.md#ip-qos){ data-preview }

- [Qos Policy](pon_advance.md#qos-policy){ data-preview } prioritizes and manages data traffic to ensure optimal performance for critical applications by controlling bandwidth, latency, and packet loss.
- [QoS Classification](pon_advance.md#qos-classification){ data-preview } is to configure the Classification Rules.
- [Traffic Shaping](pon_advance.md#traffic-shaping){ data-preview } is to configure the Traffic Shaping Rules. 

[**IPv6**](pon_advance.md#ipv6){ data-preview }  

- [IPv6 Enable/Disable](pon_advance.md#ipv6-enabledisable){ data-preview } is to enable or disable IPv6. 
- [RADVD](pon_advance.md#radvd){ data-preview } broadcasts network prefixes and routing information to hosts, enabling SLAAC (StateLess Address Auto-Configuration) without DHCPv6. 
- [DHCPv6](pon_advance.md#dhcpv6){ data-preview } is to configure DHCPv6 Server and Relay.
- [MLD Proxy](pon_advance.md#mld-proxy){ data-preview } forwards Multicast Listener Discovery (MLD) messages between IPv6 networks, acting as a simple relay for multicast group memberships across subnet boundaries. 
- [MLD Snooping](pon_advance.md#mld-snooping){ data-preview } switches listen to MLD (Multicast Listener Discovery) messages to intelligently forward IPv6 multicast traffic only to ports with interested receivers, reducing unnecessary network flooding. 
- [IPv6 Routing](pon_advance.md#ipv6-routing){ data-preview } configures fixed paths for network traffic by specifying destination prefixes and next-hop addresses, ideal for small or stable networks.
- [IP/Port Filtering](pon_advance.md#ipport-filtering){ data-preview } controls network access by blocking/allowing specific IPv6 addresses and port numbers, enhancing security.
- [IPv6 ACL](pon_advance.md#ipv6-acl){ data-preview } filters network traffic by defining permit/deny rules based on IPv6 addresses, ports, or protocols to enforce security policies.

### VII. Admin
- [GPON](pon_admin.md#gpon-settings){ data-preview } (Gigabit Passive Optical Network) is to be established and managed the router connection to, including authentication (e.g., LOID/PPPoE), VLAN tagging, and bandwidth allocation for fiber-optic broadband services.
- [OMCI](pon_admin.md#omci-information){ data-preview } short for ONU Management and Control Interface, is a protocol defined in the GPON standard. 
- [Remote Management](pon_admin.md#remote-management){ data-preview } allows external access to router settings. 
- [Reboot](pon_admin.md#reboot){ data-preview } the router to refresh its system, resolve temporary glitches, or apply critical updates, in cases like firmware updates, network instability, or scheduled maintenance. It may cause a brief network disconnection for about 1~3 minutes. 
- [Multi-lingual Settings](pon_admin.md#multi-lingual-settings){ data-preview } supports the web interface or app to display in multiple languages to improve accessibility for global users.
- [Backup/Restore](pon_admin.md#backuprestore){ data-preview } supports saving the router's current configuration (e.g., Wi-Fi passwords, port rules) to a file for backup, and later restoring it to recover settings after a reset. 
- [System Log](pon_admin.md#system-log){ data-preview } records all critical router events (e.g., connections, errors, attacks) for troubleshooting and security monitoring. 
- [Password](pon_admin.md#password){ data-preview } is the authentication credential (letters/numbers/symbols) required to access the router's admin interface, thus to prevent unauthorized changes. 
- [Firmware Upgrade](pon_admin.md#firmware-upgrade){ data-preview } updates the router's operating system to patch vulnerabilities or add features (requires stable power/internet). 
- [Time Zone](pon_admin.md#time-zone){ data-preview } configuration sets local time for accurate logs/schedules (e.g., parental controls). 
- [TR-069](pon_admin.md#tr-069){ data-preview } configuration enables remote ISP management (auto-configuration/troubleshooting).
- [Logout](pon_admin.md#logout){ data-preview } is to securely exit the admin interface to prevent unauthorized access.


### VIII. Diagnostics
- [Ping](pon_diagnostics.md#ping){ data-preview } is a network tool that sends ICMP echo requests to test if a device (e.g., server, router) is reachable and measure response time. It’s used to verify connectivity, detect latency, or troubleshoot network failures. 
- [Ping6 ](pon_diagnostics.md#ping6){ data-preview } is an IPv6 version of the ping tool that sends ICMPv6 Echo Requests to test connectivity and measure latency between devices in an IPv6 network. It's primarily used to verify IPv6 reachability, troubleshoot network issues, and assess path performance. 
- [Tracert (Traceroute)](pon_diagnostics.md#tracert){ data-preview } is a network diagnostic tool that maps the path (hops) between your router and a target IP/domain, measuring latency at each node. It's used to identify routing failures, congestion points, or connectivity issues across networks. 
- [Tracert6 ](pon_diagnostics.md#tracert6){ data-preview } is the IPv6 version of traceroute, mapping the network path (hops) to a target IPv6 address using ICMPv6 probes. It's used to diagnose routing issues, latency problems, or connectivity failures in IPv6 networks. 

### IX. Statistics
- [Interface Statistics](pon_statistics.md){ data-preview }
monitors real-time traffic data (e.g., speed, packets, errors) on router interfaces (WAN/LAN) and thus help to troubleshoot network performance or congestion. 
- [PON Statistics](pon_statistics.md#pon-statistics){ data-preview }
tracks optical signal metrics (e.g., power levels, uptime) for fiber connections (GPON/EPON), and thus to ensure stable ISP link quality. Only visible on fiber-optic routers with PON ports.
